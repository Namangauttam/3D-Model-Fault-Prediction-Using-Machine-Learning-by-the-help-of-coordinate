ðŸ“„ Project Title: Fault Detection by Coordinates in 3D Model

This project detects faults in 3D models using coordinates extracted from STL files and vibration sensor data. It utilizes supervised ML models to predict faults at specific points. Built with Python, Flask, and machine learning techniques, it enables users to upload 3D files and view predictions instantly.

1. Introduction

This project aims to automatically detect faults in 3D models by analyzing vibration sensor data and coordinate data extracted from .stl files...

2. Dataset

The dataset (ADXL345_SensorData.csv) contains:

  • X-direction: X-axis coordinate/vibration
  • Y-direction: Y-axis coordinate/vibration
  • Z-direction: Z-axis coordinate/vibration
  • Error_found: Indicates if a fault is present (0 = No fault, 1 = Fault)

3. Libraries and Technologies Used

  • Data: pandas, numpy
  • Visualization: matplotlib, seaborn
  • Modeling: scikit-learn, XGBoost, imblearn (ADASYN)
  • Web App: Flask, HTML
  • File Handling: numpy-stl, pickle

4. Data Preprocessing

  • Handled missing values and outliers
  • Used percentile method for outlier treatment
  • Balanced dataset using ADASYN

5. Exploratory Data Analysis

  • KDE plots for X, Y, Z
  • Scatter plots, violin plots, heatmaps
  • Boxplots before/after outlier treatment

6. Model Training and Selection

  • Trained Logistic Regression, Random Forest, XGBoost, Naive Bayes, KNN, SVC
  • Used GridSearchCV for hyperparameter tuning

7. Model Evaluation

  • Evaluated using Accuracy, F1 Score, ROC AUC, Classification Report
  • Selected best model and saved as model.pkl

8. Backend Deployment

  • Homepage and navigation pages
  • /predictor - Upload form
  • /predict - Reads STL, predicts, returns results

9. STL File Handling

  • STL uploaded â†’ coordinates extracted via numpy-stl
  • Predictions made using saved model
  • Uploaded files auto-deleted post-prediction

10. Result

Each 3D coordinate is labeled with:

  • Fault Detected if error predicted
  • No Fault otherwise
